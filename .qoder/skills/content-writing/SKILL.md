# 通用内容创作

> 职责：根据写作任务的复杂度，自动匹配合适的写作规则组合，产出高质量的面向读者的文字内容。

## 核心规则引用

本 Skill 基于两条互补的写作规则工作：

| 规则 | 定位 | 控制层面 |
|------|------|---------|
| `writing-style.md` | 风格手册 | **怎么写** — 语气、结构、节奏、自检 |
| `writing-process.md` | 流程手册 | **怎么做** — 定位→构思→写作→自检四阶段 |
| `title-crafting.md` | 标题手册 | **怎么起标题** — 技巧、思考路径、自检 |

执行前通过 `fetch_rules` 获取对应规则的完整内容。**凡涉及标题拟定的环节，必须加载 `title-crafting.md`。**

## 复杂度判定与模式选择

收到写作任务后，先评估复杂度，自动选择模式：

### 判定逻辑

```
是否已明确"写给谁看"和"写什么"？
  ├─ 是 → 简单模式
  └─ 否 → 是否需要严格质量管控或从零开始？
            ├─ 是 → 复杂模式
            └─ 否 → 中等模式
```

### 三档模式

| 模式 | 适用场景 | 规则组合 | 典型任务 |
|------|---------|---------|---------|
| **简单模式** | 主题明确，思路清晰 | 仅 `writing-style.md` | 短文案、已有大纲的正文、语气润色 |
| **中等模式** | 主题明确但需要组织 | `writing-process.md` 定位阶段 + `writing-style.md` 全文 | 有明确主题但需要结构化的种草文、攻略 |
| **复杂模式** | 从零开始或需要严格质量保障 | `writing-process.md` 全流程 + `writing-style.md` 在写作阶段辅助 | 系列专栏、深度长文、品牌内容 |

### 场景速查

**简单模式触发信号**：
- 用户说"帮我写一段xxx"、"润色一下"、"改个语气"
- 已有大纲/框架，只需填充正文
- 短文案（朋友圈、微博、评论区话术）

**中等模式触发信号**：
- 用户给了主题但没给结构
- 需要产出完整的社交媒体笔记（标题+正文+标签）
- 产品种草、游戏安利、体验分享

**复杂模式触发信号**：
- 用户说"帮我写一篇关于xxx的文章"但方向模糊
- 系列内容、品牌内容、需要多人审核
- 深度长文、观点输出、行业分析

## 执行流程

### 简单模式

1. **确认三要素**：快速确认写给谁、痛点、读完收获（参照 `writing-style.md` 三个必答题）
2. **拟定标题**：按 `title-crafting.md` 执行流程，产出 3-5 个候选标题供选择
3. **直接写作**：围绕选定标题，按"凤头猪肚豹尾"结构 + 语气红绿灯产出内容
4. **自检**：过 `writing-style.md` 自检清单 + `title-crafting.md` 标题自检清单
5. **输出定稿**

### 中等模式

1. **定位**：填写三要素定义表（目的/对象/场景），通过读者视角验证门禁
2. **拟定标题**：按 `title-crafting.md` 执行流程，明确内容核心后产出 3-5 个候选标题，标注使用的技巧
3. **直接写作**：围绕选定标题，按 `writing-style.md` 的结构口诀和万能公式展开
4. **自检**：过 `writing-style.md` 自检清单 + `title-crafting.md` 标题自检清单
5. **输出定稿**

### 复杂模式

1. **定位**：填写三要素定义表，通过读者视角验证门禁（`writing-process.md` 阶段一）
2. **构思**：提炼核心观点（一句话），搭建结构框架，通过溪流原则检查门禁（`writing-process.md` 阶段二）
3. **拟定标题**：基于核心观点，按 `title-crafting.md` 五条思考路径各出备选，打磨后产出 3-5 个候选标题，标注技巧与路径
4. **写作**：围绕选定标题，按框架展开正文，遵循 `writing-style.md` 的语气规范和结构口诀（`writing-process.md` 阶段三 + `writing-style.md`）
5. **自检**：完成 `writing-process.md` 九项自检清单 + `title-crafting.md` 标题自检清单，全部通过方可定稿（`writing-process.md` 阶段四）
6. **输出定稿**

## 输入要求

### 必需信息

| 信息 | 说明 |
|------|------|
| 写作主题/任务描述 | 写什么内容 |
| 目标读者 | 写给谁看（可在定位阶段补充） |

### 可选信息

| 信息 | 说明 | 默认 |
|------|------|------|
| 发布平台 | 小红书/公众号/微博/豆瓣等 | 不限 |
| 篇幅要求 | 字数或段落数 | 由内容决定 |
| 语气偏好 | 轻松/专业/文艺/幽默等 | 聊天式 |
| 参考素材 | 已有大纲、资料、灵感来源 | 无 |
| 复杂度指定 | 用户可直接指定模式 | 自动判定 |

信息不全时，通过 `AskUserQuestion` 收集缺失项。

## 输出格式

根据任务类型灵活调整，典型输出包括：

### 社交媒体笔记

```markdown
## 标题
{正式标题}

## 正文
{完整正文}

## 标签/Hashtag
{标签列表}
```

### 一般文章

```markdown
## 标题
{文章标题}

## 正文
{完整文章内容}
```

### 润色/修改

```markdown
## 修改说明
{改了什么、为什么改}

## 修改后全文
{完整修改后内容}
```

## 与桌游笔记工作流的协作

当被 `boardgame-note-workflow` 在文案撰写阶段调用时：

- 从上游 `_process/` 中读取 `创作上下文摘要.md` 获取游戏信息
- 默认使用**中等模式**（主题已由 input-collector 确定）
- 正文结构遵循 `boardgame-copywriting` 的 6 段模板（情绪开场→规则简述→核心刺激→场景列表→进阶钩子→行动号召）
- 标题按 `title-crafting.md` 执行流程拟定，产出 3-5 个候选并标注技巧
- Hashtag 按垂类/泛流量/情绪/专属四层输出

独立调用时不受上述桌游特定约束。

## 质量底线

无论哪种模式，以下底线不可突破：

1. **选题底线**：内容必须携带态度（你的看法）或发现（你的洞察），纯信息搬运不算写作
2. **接收优先**：以读者接收到的信息为准，不以作者想表达的为准
3. **聊天不说教**：平等分享，禁止居高临下
4. **溪流原则**：全文一条主线，可以拐弯但不分叉
