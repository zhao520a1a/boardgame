# 桌游实战字幕故事提炼

> 把视频字幕翻译成可直接用于文案的故事素材

---

## 输入

| 必填 | 说明 |
|------|------|
| 字幕文件 | TXT 或 SRT 格式 |
| 游戏名称 | 用于语境理解 |

---

## 处理流程

### 1. 清洗字幕

**删除**：
- 时间码、序号
- 纯规则讲解（"每回合抽一张牌..."）
- 引导/推广语（"记得点赞"）
- 视觉指代（"你看这里"）

**修正**：
- ASR 错误（同音字、专有名词）
- 断句错误
- 重复表达（保留最精炼一次）

### 2. 识别故事片段

找这六类内容：

| 类型 | 标志 | 优先级 |
|------|------|--------|
| 冲突爆发 | "你骗我""凭什么" | ★★★★★ |
| 意外转折 | "没想到""居然" | ★★★★★ |
| 情绪宣泄 | "太刺激了""完了完了" | ★★★★ |
| 社交互动 | "你信我""他坑我" | ★★★★ |
| 策略讨论 | "应该""如果" | ★★★ |
| 结果评价 | "这局太..." | ★★★ |

**提取规则**：
- 标记高价值行后，前后各扩展 2-3 句
- 相邻高价值行（间距 ≤ 3 句）合并
- 数量：500行以下 5-8 个，500-1500行 8-15 个，1500行以上 15-25 个

### 3. 活人感提炼

**三原则**：

1. **保留"我"的在场感**
   - ✅ "当时""那一刻"
   - ✅ "我翻到第三张..."
   - ❌ 避免"玩家翻到第三张..."

2. **保留情绪，删减废话**
   ```
   原始：完了完了完了完了这把要输了
   提炼：完了，这把要输了
   ```

3. **视觉转文字**
   ```
   原始：你看他那个表情 太搞笑了
   提炼：他那个表情，像中了彩票又发现过期了
   ```

**判定标准**（4 项满足 3 项）：
- 有具体数字/细节？
- 有情绪/感受词？
- 有节奏变化（短句、破折号）？
- 读出来像人在说话？

---

## 输出

### 文件 1：story-fragments.md

**只保留核心内容**：

```markdown
# {游戏名}故事素材

> 来源：{字幕文件名}

---

## {片段标题1}

> "{提炼后的文字...}"

---

## {片段标题2}

> "{提炼后的文字...}"
```

**禁止包含**：
- ❌ 角色对应表
- ❌ 统计信息
- ❌ 场景分类索引
- ❌ 与文案的映射表
- ❌ 原始位置标注
- ❌ 场景标签、情绪关键词

### 文件 2：story-narrative.md

```markdown
# 《{游戏名}》实战故事

---

{凤头：1-2句冲突/反转开场}

{猪肚：3-5个片段串联，保持情绪起伏}

{豹尾：1句话干脆收尾}
```

**要求**：
- 凤头：选★★★★★冲突/转折片段
- 猪肚：按时间线或情绪由低到高
- 过渡：1句简短衔接，不用"接下来"
- 字数：300-600字

---

## 质量底线

1. 必须有情绪波动
2. 必须有口语化表达
3. 必须有具体细节
4. 必须有人物互动
5. 不能比原始字幕还"干"

---

## 自检清单

- [ ] 读起来像真人说话，不像 AI 报告？
- [ ] 情绪词已保留？
- [ ] 视觉指代已转文字？
- [ ] 重复已精简但情感核心保留？
- [ ] 有上下文语境？
- [ ] 叙事有情绪曲线？

---

## 异常处理

| 问题 | 处理 |
|------|------|
| 字幕为空/无效 | 提示确认路径 |
| 质量极差 | 建议人工校对 |
| 纯规则讲解 | 建议用 `boardgame-rule-structure` |
| 无法分辨说话人 | 统一标为 P1 |
| 字幕过短（<50行） | 提取 3-5 个核心片段 |
