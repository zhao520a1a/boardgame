# 前置输入收集

> 职责：收集并校验桌游笔记创作所需的全部前置信息，输出标准化上下文摘要。

## 必填输入项

| 输入项 | 说明 | 示例 | 校验规则 |
|--------|------|------|---------|
| 游戏名称 | 中文名 + 英文名 | 翻转七 / Flip 7 | 必须包含中文名 |
| 核心机制 | 1-3 个关键词 | 押注博弈、概率计算 | 至少 1 个 |
| 人数/时长 | 基础参数 | 3-6人 / 15分钟 | 数字格式 |
| 游戏规则 | 完整规则文本或链接 | （用户提供） | 不可为空 |
| 参考素材 | 游戏盒照、组件照、实拍图 | 2-4 张图 | 至少 1 张 |
| 目标平台 | 发布渠道 | 小红书 | 从预设选项中选 |
| 推广目标 | 内容导向 | 涨粉吸睛 | 从预设选项中选 |

## 可选输入项

| 输入项 | 说明 | 默认值 |
|--------|------|--------|
| 风格偏好 | 文艺调性 / 高点击率 / 人物产品 | 根据推广目标自动推荐 |
| 热点节点 | 春节 / 周末 / 团建 / 情侣 / 通用 | 通用 |
| 是否需要规则图 | 是否生成一页纸规则讲解图 | 否 |

## 收集流程

### 1. 解析用户输入

检查用户消息中是否已包含部分信息，已提供的直接提取，未提供的标记为待收集。

### 2. 补充缺失项

对缺失的必填项，通过 `AskUserQuestion` 逐项收集：

**平台与目标**（如未指定）：
- 目标平台：小红书 / 豆瓣 / B站
- 推广目标：涨粉吸睛 / 种草安利 / 带货转化

**风格推荐逻辑**（如未指定风格偏好）：

| 推广目标 | 推荐风格线 |
|---------|-----------|
| 涨粉吸睛 | B 线（高点击率） |
| 种草安利 | A 线（文艺调性） |
| 带货转化 | C 线（人物产品） |

### 3. 规则文本预处理

收到规则文本后进行结构化提取：

```
- 游戏主题/背景
- 获胜条件
- 核心回合流程
- 关键行动选项
- 特殊规则/边界情况
```

### 4. 输出标准化上下文摘要

收集完成后，输出以下格式的上下文摘要：

```markdown
## 创作上下文摘要

**游戏名称**：《中文名》/ English Name
**核心机制**：关键词1、关键词2
**人数/时长**：X-Y人 / Z分钟
**目标平台**：平台名
**推广目标**：目标描述
**风格线**：A/B/C + 子风格
**热点节点**：节点名称
**规则图**：需要/跳过

### 规则摘要
- 主题：...
- 获胜条件：...
- 核心流程：...
- 关键行动：...
- 特殊规则：...

### 参考素材
- 素材1描述
- 素材2描述
```

## 校验清单

输出前逐项确认：

- [ ] 游戏名称包含中英文
- [ ] 规则文本已结构化提取
- [ ] 至少有 1 张参考素材
- [ ] 风格线已确定
- [ ] 上下文摘要格式完整
