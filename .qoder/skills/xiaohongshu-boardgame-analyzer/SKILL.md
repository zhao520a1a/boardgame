# 小红书桌游内容分析与选题引擎

> 职责：作为桌游创作前的"市场调研环节"，通过采集和分析小红书桌游垂类热门内容，输出数据驱动的分析报告和可复用的爆款选题库，为后续文案撰写和发帖策略提供依据。

---

## 一、核心规则引用

本 Skill 执行时需加载以下规则文件：

| 规则 | 定位 | 加载时机 |
|------|------|---------|
| `xiaohongshu-content-analysis.md` | 采集与分析规范 | 阶段 1-2：数据采集和内容分析时 |
| `topic-library-standards.md` | 选题库标准 | 阶段 4：选题库构建时 |

执行前通过 `fetch_rules` 获取对应规则的完整内容。

## 二、输入要求

| 输入项 | 必填 | 说明 | 默认值 |

### 阶段 1：数据采集

#### 路径 A：浏览器自动化采集（默认）

**Step 1.1：打开小红书搜索页**

```
操作：mcp__browser-use__new_page(url="https://www.xiaohongshu.com")
等待：页面加载完成
检测：mcp__browser-use__take_snapshot()
判断：
├─ 检测到登录弹窗 / 遮罩层
│   → 提示用户："检测到小红书需要登录，请在浏览器中手动登录后告诉我继续"
│   → 等待用户确认后重新 take_snapshot
└─ 页面正常加载 → 继续
```

**Step 1.2：执行搜索**

```
操作：
1. mcp__browser-use__take_snapshot() — 定位搜索框
2. mcp__browser-use__click(uid=搜索框uid) — 聚焦搜索框
3. mcp__browser-use__fill(uid=搜索框uid, value="[关键词]")
4. mcp__browser-use__press_key(key="Enter")
5. 等待 3 秒让结果加载
6. mcp__browser-use__take_snapshot() — 确认搜索结果已加载

异常处理：
├─ 搜索框未找到 → 尝试直接导航到搜索URL
│   mcp__browser-use__navigate_page(url="https://www.xiaohongshu.com/search_result?keyword=[编码后关键词]")
└─ 搜索结果为空 → 提示用户更换关键词
```

**Step 1.3：滚动加载更多结果**

```
循环 [采集数量 / 10] 次（每次滚动约加载 10 条）：
  1. mcp__browser-use__press_key(key="End") — 滚动到底部
  2. 等待 2 秒（遵循采集频率限制）
  3. mcp__browser-use__take_snapshot() — 检查是否有新内容加载

终止条件：
├─ 达到目标采集数量
├─ 连续 2 次滚动无新内容加载
└─ 检测到"没有更多了"提示
```

**Step 1.4：列表页数据提取**

```
操作：mcp__browser-use__take_snapshot(verbose=true)

从 snapshot 中解析每条笔记：
├── 标题文本 — 笔记卡片中的标题
├── 点赞数 — 数值提取，处理"万"单位（1.2万→12000）
├── 作者昵称 — 笔记卡片中的作者名
├── 笔记链接 — 卡片的链接地址
└── 内容类型 — 图文/视频标识

数据清洗：
├── 去除广告/推广笔记（标记为"赞助""广告"的条目）
├── 去重（同一笔记可能在不同位置出现）
└── 数值标准化（"万"→具体数字）
```

**Step 1.5：详情页抽样采集（TOP 5-10 高互动笔记）**

```
排序：按列表页点赞数降序，取前 5-10 条

对每条高互动笔记：
  1. mcp__browser-use__new_page(url=笔记链接)
  2. 等待 2 秒
  3. mcp__browser-use__take_snapshot() — 提取详情数据：
     ├── 完整正文内容
     ├── Hashtag 列表
     ├── 收藏数
     ├── 评论数
     ├── 发布时间
     ├── 图片数量
     └── 评论区前 5 条热评（如可见）
  4. mcp__browser-use__take_screenshot(filePath=截图路径) — 存档

完成后关闭详情页标签页，返回搜索结果页
```

**Step 1.6：截图存档**

```
mcp__browser-use__take_screenshot(fullPage=true, filePath="_process/search_screenshot.png")
```

#### 路径 B：降级手动输入

**触发条件**（任一满足）：

```
├── 浏览器工具调用失败（超时/报错/工具不可用）
├── 检测到登录墙且用户未处理（等待超过 60 秒）
├── 检测到验证码
├── 页面结构变化导致 snapshot 无法识别笔记列表
├── 连续 3 次操作失败
└── 用户主动选择手动模式（采集模式="手动"）
```

**降级流程**：

```
1. 告知用户降级原因：
   "自动采集遇到 [具体原因]，切换到手动输入模式。"

2. 引导用户提供数据（AskUserQuestion）：
   "请通过以下方式提供小红书搜索数据：

    方式 A：截图
    在小红书搜索「[关键词]」，截取搜索结果页面（至少包含 10 条笔记），
    将截图发给我。

    方式 B：文本粘贴
    在小红书搜索后，把看到的笔记信息按以下格式粘贴给我：
    标题 | 点赞数 | 收藏数 | 评论数
    （每行一条，至少 10 条）

    方式 C：混合
    截图 + 文本均可，我会合并分析。"

3. 接收用户数据：
   ├── 截图 → 读取图片，提取文字和数据（AI 视觉能力）
   ├── 文本 → 直接解析结构化数据
   └── 混合 → 合并两种来源

4. 数据质量检查：
   ├── 有效笔记 ≥ 10 条 → 正常进入分析
   ├── 有效笔记 5-9 条 → 标注"小样本分析"，继续
   └── 有效笔记 < 5 条 → 提示用户补充更多数据
```

### 阶段 2：内容深度分析

> 加载规则：`fetch_rules("xiaohongshu-content-analysis")`

#### 2.1 标题特征分析

```
分析项目：
├── 标题长度分布 — 统计字符数分布（均值、中位数、最大最小值）
├── 标题公式分类 — 按规则文件中的识别标准分类每条标题：
│   ├── 数字型：含阿拉伯数字 + TOP/必/推荐等词
│   ├── 反转型：含转折词（但是/结果/打脸/没想到）
│   ├── 悬念型：含问号/省略号/反转词（竟然/居然）
│   ├── 共鸣型：含"你""我们" + 痛点场景词
│   └── 其他：不符合以上类型
├── 高频关键词 — 分词后统计 Top 20（剔除停用词）
├── Emoji 使用 — 统计使用频率和常见 Emoji
└── 标题公式 × 互动率 — 各公式类型的平均互动数对比
```

#### 2.2 互动数据分析

```
分析项目：
├── 各指标分布统计
│   ├── 点赞：均值、中位数、P80
│   ├── 收藏：均值、中位数、P80（如有）
│   └── 评论：均值、中位数、P80（如有）
├── 综合互动分计算 — 按规则：点赞×1 + 收藏×2 + 评论×3
├── 高互动阈值确定 — P80 分位值
├── 高互动内容特征提取 — TOP 20% 笔记的共同点
│   ├── 标题公式偏好
│   ├── 内容类型偏好（图文 vs 视频）
│   ├── 标题关键词特征
│   └── 发布时间规律（如有）
└── 高互动 vs 低互动对比 — 特征差异摘要
```

#### 2.3 关键词与标签分析

```
分析项目：
├── 标题关键词词频 Top 20
│   ├── 关键词
│   ├── 出现次数
│   └── 所在标题的平均互动数
├── Hashtag 使用频率排名 Top 15（详情页数据）
├── 标签组合模式分析
│   ├── 垂类标签（桌游/棋牌类）占比
│   ├── 泛流量标签（聚会/好物类）占比
│   ├── 情绪标签（上头/好玩类）占比
│   └── 常见搭配组合
└── 热门标签与互动率的关联
```

#### 2.4 情感倾向分析

```
分析项目：
├── 标题情感分布
│   ├── 正面（推荐/好评/兴奋）
│   ├── 中性（介绍/测评/教学）
│   └── 负面（吐槽/避雷/踩坑）
├── 情感倾向 × 互动率关联
└── 评论区情感趋势（如有详情页数据）
    ├── 正面反馈关键词
    └── 用户关注点提取
```

#### 2.5 受众行为分析

```
分析项目：
├── 内容形式偏好 — 图文 vs 视频占比
├── 图片数量偏好 — 图文笔记的平均图片数
├── 高收藏内容特征 — "实用性"信号（教程/攻略/清单）
├── 高评论内容特征 — "共鸣/争议"信号（提问/投票/对比）
├── 高点赞内容特征 — "认同/有趣"信号（好笑/共鸣/种草）
└── 受众画像推断
    ├── 主力人群标签（学生/上班族/家庭/情侣等）
    ├── 内容消费偏好
    └── 购买决策关注点
```

## 四、与桌游笔记工作流的协作

### 4.1 独立调用

直接使用 `/xiaohongshu-boardgame-analyzer`，无需前置阶段。

### 4.2 被编排器调用

当被 `boardgame-orchestrator` 在创作流程中调用时：

- 如果已有 Step 1 的上下文摘要，从中提取游戏名作为搜索关键词之一
- 分析报告中的策略建议自动传递给下游：
  - `boardgame-copywriting`：热门标题公式 + 高频关键词 + Hashtag 推荐
  - `boardgame-posting-strategy`：发布时间建议 + 互动数据基准线
- 选题库可供用户在文案阶段选择创作方向

### 4.3 输出供其他 Skill 引用

其他 Skill 可从 `_process/` 中读取分析数据：
- `analysis_report.md` — 完整分析报告
- `topic_library.md` — 选题库和创作思路

## 五、质量底线

输出前逐项确认：

- [ ] 分析基于实际采集数据，非凭空推测（每个结论标注数据来源）
- [ ] 互动数据已做"万"单位转换，数值准确
- [ ] 标题公式分类遵循 `xiaohongshu-content-analysis.md` 规则
- [ ] 选题库每个选题包含：标题模板 ≥ 2、成功要素 ≥ 3、创作思路 ≥ 2 角度
- [ ] 策略建议可执行、可验证，不是空洞方向
- [ ] 降级模式下明确标注"小样本分析"和数据来源
- [ ] 报告格式规范，Markdown 表格对齐，无乱码
- [ ] 选题库选题之间有差异化，不重复同一角度

## 六、多轮交互设计

### 6.1 首轮输出结尾

完成报告和选题库后，追加交互引导：

```markdown
---

**分析完成！接下来你可以：**
- 🔍 **深挖某个维度**："标题分析再详细讲讲" / "高互动内容展开分析"
- 📝 **选择选题开始创作**："用选题3来写一篇笔记" → 自动衔接 boardgame-copywriting
- 🔄 **换个关键词分析**："再分析一下'双人桌游'" → 重新执行阶段 1-4
- 📊 **对比分析**："对比'桌游推荐'和'聚会桌游'的差异"
- ➕ **补充数据**："我再提供一些笔记数据" → 合并分析
```

### 6.2 多轮指令响应

| 用户指令 | 执行动作 |
|---------|---------|
| "XX维度详细分析" | 展开该维度的深度分析 |
| "用选题N创作" | 将选题传递给 boardgame-copywriting |
| "换关键词" | 重新执行完整流程 |
| "补充数据" | 合并新数据重新分析 |
| "更新选题库" | 基于新分析追加选题 |
