# 选题库数据结构与质量标准

## 适用范围

需要由用户手动指定选择该规则，适用于 `xiaohongshu-boardgame-analyzer` 技能在构建爆款选题库时的数据结构规范和质量标准。

---

## 一、选题质量标准

### 1.1 单条选题必备字段

每个选题必须包含以下内容，缺一不可：

| 字段 | 最低要求 | 说明 |
|------|---------|------|
| 选题名称 | 1 个 | 简洁概括选题方向（5-15 字） |
| 推荐标题公式 | 1 种 | 悬念型/数字型/共鸣型/反转型 |
| 标题模板 | ≥ 2 个 | 含 `[游戏名]` 等变量，可直接套用 |
| 核心卖点 | 1-2 个 | 选题的核心吸引力 |
| 适用游戏类型 | ≥ 1 种 | 具体游戏或类型 |
| 目标受众 | 1 个 | 受众画像描述 |
| 成功要素 | ≥ 3 个 | 该选题成功所需的关键条件 |
| 创作思路 | ≥ 2 个角度 | 差异化创作方向 |
| 最佳发布节点 | 1 个 | 推荐发布时间 |
| 推荐 Hashtag | 1 组 | 标签组合建议 |
| 互动潜力 | 1 级 | 高/中/低 |
| 数据依据 | 1 条 | 基于分析报告的哪些发现 |

### 1.2 质量红线

- 选题必须有数据依据，禁止凭空编造（每个选题标注来源分析数据）
- 标题模板必须可直接套用（变量用 `[方括号]` 标记，替换后即可使用）
- 创作思路必须具体（不接受"从用户角度出发"等空洞描述，需明确内容重点）
- 同一类别内选题之间必须有差异化（不同角度、不同受众、不同公式）

### 1.3 互动潜力评级标准

| 评级 | 判定条件 |
|------|---------|
| **高** | 选题方向在分析数据中对应 TOP 20% 高互动内容，且同类竞品较少 |
| **中** | 选题方向在分析数据中对应中等互动水平，或同类竞品较多但仍有差异化空间 |
| **低** | 选题方向较为小众，但有特定受众价值，适合垂类深耕 |

## 二、分类标准

### 2.1 桌游子领域五大类

| 一级分类 | 二级分类 | 典型游戏举例 | 受众特征 |
|---------|---------|-------------|---------|
| 策略桌游 | 德式策略 | 璀璨宝石、卡坦岛、农场主 | 核心桌游玩家，追求策略深度 |
| 策略桌游 | 美式冒险 | 幽港迷城、方舟动物园 | 偏好叙事和沉浸感 |
| 家庭桌游 | 亲子互动 | 卡斯卡迪亚、多米诺王国 | 家长+儿童，强调教育和互动 |
| 家庭桌游 | 全家同乐 | UNO、大富翁、飞行棋 | 全年龄段，低门槛 |
| 派对桌游 | 欢乐聚会 | 狼人杀、阿瓦隆、画物语 | 年轻人聚会，强调社交和欢乐 |
| 派对桌游 | 破冰社交 | 行动代号、机密暗号 | 初次见面/团建场景 |
| 双人桌游 | 情侣对战 | 七大奇迹对决、拼布艺术 | 情侣约会，轻竞技 |
| 双人桌游 | 合作通关 | 禁闭岛、瘟疫危机 | 搭档合作，强调默契 |
| 儿童桌游 | 益智启蒙 | HABA 系列、Orchard Toys | 低龄儿童，家长选购 |
| 儿童桌游 | 逻辑思维 | 拉密、智蔻 | 学龄儿童，培养逻辑能力 |

### 2.2 分类规则

- 每个子领域生成 3-5 个选题
- 总计 15-25 个选题
- 如果分析数据显示某子领域内容极少（< 3 条相关笔记），该领域选题可减至 1-2 个，并标注"数据有限"
- 跨子领域的选题归入最主要的类别，不重复收录

## 三、标题模板规范

### 3.1 变量定义

| 变量 | 说明 | 示例 |
|------|------|------|
| `[游戏名]` | 桌游中文名 | 璀璨宝石 |
| `[人数]` | 适合游玩人数 | 2-4 |
| `[N]` | 数字占位符 | 5、10、TOP3 |
| `[场景]` | 使用场景 | 聚会、约会、周末 |
| `[痛点]` | 目标受众痛点 | 无聊、冷场、不知道玩什么 |
| `[结果]` | 意外结果 | 玩到凌晨、连开三局 |

### 3.2 模板质量要求

- 模板长度：10-20 字（适配小红书标题显示）
- 必须包含至少 1 个变量（确保可复用）
- 替换变量后的标题应自然流畅，无语法问题
- 同一选题的 2 个模板应使用不同的标题公式或角度

## 四、更新维护规范

### 4.1 累积机制

- 选题库文件支持累积：多次分析结果可合并到同一选题库
- 新选题追加到对应子领域末尾
- 选题编号在类别内顺序递增

### 4.2 时效性标注

- 基于 > 90 天前数据生成的选题标注 `⏳ 待更新`
- 用户可通过重新分析更新选题库
- 更新时保留历史选题作为参考，在标题后追加 `(v1)` `(v2)` 标记

### 4.3 反馈迭代

- 用户使用某选题创作后，可标注实际互动数据作为反馈
- 反馈数据用于校准互动潜力评级
- 高于预期的选题上调评级，低于预期的标注"需复盘"
