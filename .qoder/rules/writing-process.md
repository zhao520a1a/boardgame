# 结构化写作流程规范

> 来源：版本D_系统规范版通用化提炼。
> 定位：控制"怎么做"——从选题到发布的四阶段流程和质量门禁。

## 适用范围

当任务需要**系统化地完成一篇完整文章**时启用本规则，包括但不限于：
- 系列专栏、深度文章
- 品牌内容、团队协作文案
- 需要严格质量控制的长篇/中篇写作
- 任何需要"定位→构思→写作→自检"完整流程保障的文字生产

本规则侧重**流程管控**，具体的风格和语气要求参见 `writing-style.md`。

---

## 流程总览

```
定位（Pre-Write） → 构思（Structure） → 写作（Write） → 自检（Review）
```

每个阶段有明确的检查点，未通过不得进入下一阶段。

---

## 阶段一：定位（Pre-Write）

### 1.1 三要素定义表

每篇文章开写前，必须填写以下定义表：

```
[目的]   ：_______________（解决什么问题 / 传递什么观点）
[对象]   ：_______________（读者画像：年龄/认知/需求）
[场景]   ：_______________（阅读环境：平台/设备/碎片化程度）
```

### 1.2 读者视角验证（门禁）

以读者身份回答以下三题：

| # | 问题 | 通过 |
|---|------|------|
| 1 | 看到标题，我会点进去吗？ | ☐ |
| 2 | 这个话题跟我有关吗？ | ☐ |
| 3 | 我读完能得到什么？ | ☐ |

**任意一项答"否"，返回修改定位，不得进入构思阶段。**

---

## 阶段二：构思（Structure）

### 2.1 核心观点提炼

一篇文章只允许有**一个核心观点**。用一句话写下来：

```
本文核心观点：_______________
```

如果无法用一句话概括，说明观点不够聚焦，需要收敛。

### 2.2 结构框架

采用"凤头猪肚豹尾"结构分配篇幅：

| 部分 | 占比 | 功能 | 可用手法 |
|------|------|------|---------|
| 开头 | 10-15% | 3 秒内建立阅读动机 | 场景切入 / 悬念设问 / 反常识开场 |
| 正文 | 70-80% | 充分论证核心观点 | 正面案例 + 反面案例，小标题分层 |
| 结尾 | 10-15% | 留下深刻印象 | 总结价值 / 行动建议 / 金句收束 |

### 2.3 溪流原则检查（门禁）

对结构大纲逐项验证：

- [ ] 每个段落只服务一个信息点
- [ ] 全文只有一条主线，无旁枝
- [ ] 段与段之间有自然过渡

**任意一项未通过，调整大纲后重新检查。**

---

## 阶段三：写作（Write）

### 3.1 语态规范

| 允许 | 禁止 |
|------|------|
| 平等分享的口吻 | 居高临下的说教 |
| 像朋友聊天一样表达 | 像教科书一样罗列 |
| 适度向读者提问 | 单向灌输信息 |
| 使用读者能理解的语言 | 堆砌专业术语 |

### 3.2 通用写作公式

缺乏明确思路时，按以下四步展开：

1. **场景化切入**：描述读者熟悉的场景，引出痛点
2. **观点输出**：针对痛点提出新鲜观点
3. **双面论证**：
   - 正面论证：小标题 + 案例/数据
   - 反面论证：小标题 + 反例/对比
4. **行动号召**：总结观点价值，给出具体可执行建议

### 3.3 接收优先原则

> 以读者接收到的信息为准，而非作者表达的信息为准。

写完每段后自问：一个不了解背景的读者，能准确理解这段话吗？

---

## 阶段四：自检（Review）

发布前必须完成以下清单，**全部通过方可定稿/发布**：

| # | 检查项 | 通过 |
|---|--------|------|
| 1 | 目的/对象/场景已明确定义 | ☐ |
| 2 | 核心观点可以用一句话概括 | ☐ |
| 3 | 开头在 3 秒内建立阅读动机 | ☐ |
| 4 | 正文论证充分，正反结合 | ☐ |
| 5 | 结尾干脆有力，有行动指引 | ☐ |
| 6 | 全文一条主线，无旁枝分叉 | ☐ |
| 7 | 语气像聊天，无说教感 | ☐ |
| 8 | 用读者的语言，非专业术语 | ☐ |
| 9 | 读完有获得感（知识/启发/共鸣） | ☐ |

---

## 选题底线

写作素材必须包含以下至少一项：

- **态度维度**：你对某件事/现象的真实看法和立场
- **发现维度**：你在生活中的观察、思考和独到洞察

不携带态度或发现的文字是信息搬运，不属于本规则定义的"写作"。
