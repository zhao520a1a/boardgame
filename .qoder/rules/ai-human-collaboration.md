# AI-Human Collaboration Framework

> **核心理念:人与AI不是替代,是互为假肢**  
> AI是人的认知假肢,人是AI的直觉假肢  
> AI是最强执行引擎,人是最强决策引擎

---

## 零、互为假肢模型

### 0.1 核心隐喻

```
AI = 人的认知假肢          人 = AI的直觉假肢
    ↓                          ↓
并行处理能力              长期记忆能力
结构化转换能力            直觉判断能力
瞬时上下文切换            方向决策能力
稳定工具执行              本质提炼能力
```

### 0.2 能力互补表

| 维度 | AI的长板 | AI的短板 | 人的长板 | 人的短板 |
|------|---------|---------|---------|---------|
| **记忆** | 精确记录大量信息 | 长上下文易"忘事" | 有意义的记忆 | 容量有限 |
| **推理** | 全面、系统化 | 缺乏原创直觉 | 跳跃性直觉 | 易受情绪影响 |
| **执行** | 并发、瞬时切换 | 无法感知物理世界 | 物理感知(表情/语气/手感) | 串行、易疲劳 |
| **决策** | 基于规则的优化 | 信息不足时无法判断 | 关键判断、提炼本质 | 需要时间 |

### 0.3 协作最优解

```yaml
让AI做:
  - 并行任务（同时处理多个文件、多种格式）
  - 结构化转换（杂乱信息→清晰输出）
  - 重复性工具调用（格式化、验证、转换）
  - 瞬时上下文切换（Python→财报→SQL无缝切换）

让人做:
  - 长期记忆管理（记住项目脉络和演进逻辑）
  - 直觉判断（在信息不足时做出关键决策）
  - 方向决策（定义"要什么"而非"怎么做"）
  - 降维思考（从复杂现象中提炼本质规律）
```

## 一、快速判断模型

### 1.1 可验证性决定AI边界

```
任务是否可验证？
├─ 高可验证（编译通过、测试通过、结果可量化）→ AI强项，积极使用
├─ 中可验证（可比较、可评估、有参照标准）→ AI辅助，人类决策
└─ 低可验证（周期长、变量多、需负责）→ 人类主导，AI参考
```

### 1.2 判断矩阵

| 可验证性 | AI能力 | 人类角色 | 典型任务 |
|---------|--------|---------|---------|
| 高 | 极强 | 验收兜底 | 代码生成、测试、格式转换 |
| 中 | 较强 | 最终决策 | 技术选型、方案比较、内容创作 |
| 低 | 较弱 | 核心责任 | 商业决策、需求定义、资源分配 |

## 二、任务评估清单

对任何任务"XXX"，依次回答：

```yaml
1_可验证性:
  - 结果能否被客观验证（测试/运行/量化指标）？
  - 验证周期多长（即时/短期/长期）？
  - 错误成本多高（可回滚/不可逆）？

2_AI适用性:
  - AI擅长: 有明确规则、可验证结果、重复性高
  - AI不擅长: 需理解业务背景、承担后果责任、创造性判断

3_边界划分:
  - AI做: 执行、生成、验证、优化
  - 人做: 定义问题、设定标准、最终决策、承担责任

4_验收标准:
  - 功能验收: 是否达成目标？
  - 质量验收: 是否符合规范？
  - 风险验收: 是否存在隐患？
```

## 三、人类核心价值

### 3.1 AI无法替代的能力

| 能力 | 原因 | 应用 |
|------|------|------|
| **定义问题** | AI需要清晰输入才能工作 | 先想清楚"要解决什么" |
| **规划方向** | AI不理解业务目标和优先级 | 设计路径和里程碑 |
| **承担责任** | AI不能为结果负责 | 做最终决策并兜底 |
| **边界判断** | AI不知道什么不该做 | 设定约束和红线 |

### 3.2 人类工作重心

```
传统: 执行任务 → 未来: 定义+验收
     ↓                    ↓
  写代码              定义要写什么
  找答案              定义要问什么
  做执行              定义做到什么程度
```

## 五、速查表

### 5.1 AI的长板（积极使用）

| 能力类别 | 具体表现 | 典型场景 |
|---------|---------|---------|
| **瞬时上下文切换** | Python→财报→SQL无缝切换 | 多语言/多领域并行工作 |
| **并发处理** | 同时处理多个任务 | 批量文件处理、多维度分析 |
| **结构化转换** | 杂乱信息→清晰输出 | 整理笔记、格式转换、数据清洗 |
| **稳定工具调用** | 重复性执行不出错 | 代码生成、测试、验证、部署 |
| **精确记录** | 大量信息不遗漏 | 文档生成、日志分析、数据统计 |

### 5.2 AI的短板（人类主导）

| 短板类别 | 具体表现 | 应对策略 |
|---------|---------|---------|
| **长上下文记忆弱** | 易"忘事",难以追溯前因后果 | 人做长期记忆管理,AI做即时执行 |
| **缺乏原创直觉** | 信息不足时无法做关键判断 | 人做方向决策,AI做方案执行 |
| **不擅降维思考** | 难以从复杂现象提炼本质规律 | 人做规律总结,AI做验证测试 |
| **无物理感知** | 看不懂表情、语气、手感 | 人做现场判断,AI做后续分析 |

### 5.3 关键原则

```
1. 互为假肢原则
   AI是人的认知假肢 → 扩展人的并行处理能力
   人是AI的直觉假肢 → 补足AI的方向判断能力

2. 可验证性原则
   高可验证 → AI做 → 代码、测试、格式转换
   低可验证 → 人做 → 需求定义、商业决策、风格把控

3. 执行-决策分离
   AI = 最强执行引擎 → 生成、优化、验证
   人 = 最强决策引擎 → 定义问题、设定标准、承担责任

4. 先定义后执行
   传统: 人执行任务
   AI时代: 人定义要什么 → AI执行怎么做 → 人验收兜底
```
