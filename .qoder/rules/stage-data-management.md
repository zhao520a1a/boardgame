# 多阶段工作流数据管理

## 适用范围

需要由用户手动指定选择该规则，适用于由多个阶段协作完成的任务。每个阶段的数据独立管理，阶段文件夹位于版本文件夹（`v{N}_{MMDD}/`，参见 `version-control.md`）内部。

## 文件夹命名

```
{NN}_{stage-keyword}/
```

- **NN**：两位序号（01, 02, 03...），按执行顺序排列
- **stage-keyword**：英文小写、连字符分隔，与阶段职责对应

## 目录结构

每个阶段采用**结果直出 + 过程收纳**的两层结构：

```
{NN}_{stage-keyword}/
├── result-a.png            # 最终交付物
├── result-b.md             # 最终交付物
├── stage_summary.md        # 阶段摘要（必须）
└── _process/               # 过程数据
    ├── upstream-input.md   # 上游传入 / 用户补充 / 草稿迭代 / 反馈记录
    └── ...
```

**根部** — 仅放最终交付物和 `stage_summary.md`。
**`_process/`** — 其余一切（上游传入、用户补充输入、思考规划、草稿迭代、修改反馈等）。

## 跨阶段数据传递

- 下游需要上游结果时，将上游根部文件**复制**到下游 `_process/` 中
- 各阶段文件夹保持自包含，不依赖其他阶段的路径
- 工作流 Skill 应明确定义阶段间的传递清单

## stage_summary.md

每个阶段文件夹必须包含：

```markdown
# {阶段名称} 执行摘要

- 状态：✅ 已完成 / ⏸️ 暂停 / ❌ 跳过
- 关键决策：{要点}
- 主要输出：{文件列表及说明}
- 传递下游：{传递文件列表}
- 备注：{问题 / 待优化项}
```

## workflow_summary.md

版本文件夹根目录必须包含，记录全局进度与交付物：

```markdown
# {任务名称} 工作流记录

- 版本：v{N}
- 日期：{YYYY-MM-DD}

| 阶段 | 状态 | 输出摘要 |
|------|------|---------|
| 01 xxx | ✅ | ... |
| 02 xxx | ❌ 跳过 | {原因} |

## 最终交付清单

| 文件路径 | 说明 |
|---------|------|
| {NN}_{stage}/{filename} | ... |
```

## 约束

1. **结果直出**：最终交付物放阶段根部，不可藏入 `_process/`
2. **过程收纳**：非交付数据一律放入 `_process/`，不可散落根部
3. **摘要必备**：每个阶段含 `stage_summary.md`；版本根目录含 `workflow_summary.md`
4. **自包含传递**：跨阶段传递通过文件复制，禁止路径引用
5. **跳过留痕**：被跳过的阶段仍建文件夹，仅含 `stage_summary.md` 标记「❌ 跳过」及原因
